# 小旭恐惧指数 (XiaoXu Fear Index, XXFI)

## 一、这是什么

一个**市场情绪反向指标**。灵感来自朋友"小旭" 2024-2026 的真实炒股实录：她反复"卖飞（一卖就起飞）""买在山顶（一买就被套）""恐慌割肉在最低部"。

实证统计（A股用通达信日K、港股用 akshare 新浪源日K核验，20+ 笔具名操作）：
- **卖飞率 50%**（12 次卖出中 6 次，卖后 5 日最高涨幅 ≥8%，幅度可达 +27.8%）；
- **买即套率 86%**（7 次买入中 6 次买在山顶/追涨被埋，幅度可达 −33%/−26%/−25%/−22%；含 3 笔港股陷阱：澜起 HK6809 −26%、开拓 HK09939（听大神荐股、自择时）两次买入 5/7@2.94→−25% 与 6/2@2.80→−22%）；**但唯一的"干净买入"是英唐智控(300053, 4/29@16.77 买在暴跌后阶段底，15日最高 +21.9% 且从未破买入价)——而她恰恰在这只票上反复卖飞(5/12/7/9)，证明病灶在卖点而非买点。**
- **真恐慌割肉** 2 次（连跌约 −15% 后卖出，卖后均反弹 +9%~+24%）；
- **真追高买山顶** 多发生在连续大涨后（博杰 100% 窗口分位、光电前 5 日 +8.9%）。

**结论**：小旭的"恐惧"经常出现在市场阶段底部，"贪婪"经常出现在阶段顶部。因此把市场当下的恐惧/贪婪程度量化，即可作为**反向参考**——越恐惧越该买，越贪婪越该卖。

### 旗舰案例：胜宏科技(HK 02476) —— 一只票演全三种亏钱模式（今年最大亏损）

港股胜宏科技（2026-04-21 上市，A+H，AI/高算力 PCB 高波动新股），akshare 港股日K核验，是 XXFI 反向逻辑最典型的单票验证：

1. **卖飞**：5/21 恐慌割肉~20多万 @346.2，**次日 +16.6%**，5/28 冲到 438.2 → 卖后最高 **+26.6%**；她 5/22 盘中说"都14个点了、人麻了"与真实盘中 +14% 吻合。
2. **越跌越买被埋**：5/28 见顶后回落，她 6/24 减仓至三成，却 6/25@367.6、6/26@342.4 **越跌越买加回**；随后 7/2 暴跌 -16.3%，一路到 7/10 最低 228.4，较 6/26 买点 **-33%**。
3. **追反弹被骗（不记打）**：前几日清仓后，7/9 反弹 +6.5% 诱多，她 7/10 追进场 → 当日 高275→**收230.2(-12.87%)创上市新低**，盘中振幅 -16.9%。

> **行为标记：复读式单票追涨杀跌 / 报复性回补。** 她对同一只让她亏钱的强势票反复进出、"不记打"，情绪锚定极强——这类"越亏越上头"的单票行为，是散户反向信号里纯度最高的来源。

### 其他已核验港股案例（akshare 新浪源日K 复核）

- **澜起科技(HK 6809，2026-02-09 港交所主板上市，发行价 106.89)**：2026-05-27 开盘追高买入 @469（开盘 484.6、盘中最低 455 一路下跌，与实录"开盘买入当天一路下跌"吻合）。该股自 IPO 后暴涨至 ~500，她买在**暴涨后 87% 分位高位**；随后 5/29 单日 -12.8%、6/10 最低 345.2，**较买入 -26.4%**。典型"买在暴涨后山顶"陷阱。
- **开拓药业-B(HK 09939)**：**整体是听网络大神推荐**（"他推荐的票都涨了"），但**两次买点都是小旭自己找的**，两次均被埋（akshare 新浪源核验）：
  - ① 2026-05-07 @2.94（14:34）自找买点；前一日 5/5 刚 +10.3% spike 后两日回落，她接在回落中途，随后阴跌至 5/28 最低 2.21，**较买入 -24.8%**。
  - ② 2026-06-02 @2.80（9:43）自找买点（票本身是听大神推荐）；买在当日高 2.82 附近（前一日 6/1 +8.7%、当日收 2.66 低于买价），随后阴跌至 6/23 最低 2.18，**较买入 -22.1%**。
  - → **选股交给大神、买卖点自己定却总选在高位回落前** = "听消息 + 自择时"陷阱：亏钱的源头在别人的选股，亏损的幅度在自己糟糕的择时，比单纯追涨更隐蔽。

> ⚠️ 反向指标的局限：仅在"情绪极端 + 趋势反转"时有效，震荡市会频繁假信号。务必与自身交易系统、止损纪律配合使用，切勿单独作为买卖依据。

## 二、指标定义

- **XXFI（头条指数，0-100）**：越大 = 市场越恐惧。≥75 极度恐惧，≤25 极度贪婪。
- **GreedIndex（贪婪指数，0-100）**：越大 = 市场越贪婪。
- **反向信号**：按 XXFI 绝对区间判定（非"恐惧分 vs 贪婪分"相对比较），共五档：`BUY`（≥75 极度恐惧，反向买入）/ `ACCUMULATE`（60–75 恐惧，逢低吸纳）/ `HOLD`（40–60 中性）/ `REDUCE`（25–40 偏贪婪，逢高减仓不追涨）/ `SELL`（<25 极度贪婪，反向卖出）。**注意：XXFI 越低=恐惧越低=相对越贪婪，低分区应判 REDUCE/SELL，不要误判为 BUY。**

### 分项计算（详见 xiaoxu_fear_index.py）

恐惧分（4 项加权）：
- `drawdown`：宽基指数（默认上证指数）近 20 日最大回撤，×500（8%→40，16%→80）
- `breadth`：下跌家数/上涨家数，(r−0.5)×100（1.0→50，1.5→100）
- `limitdown`：跌停/涨停比，×50（1→50，2→100）
- `vol`：当前 20 日波动率历史分位 ×100。`vol_pct` 的**分位窗口可调**：默认取近 260 日（看全年相对水平）；设 `vol_window=60` 看近 60 日"纯情绪"相对水平，对近期波动更敏感、可剔除长期低波动基线造成的整体偏高。实测两种窗口对本周(2026-07-06~10)均给出 83%+ 高位 → **当前高波动是真实持续状态，并非长窗口 artifact**（详见下方"波动率窗口的经验"）。

贪婪分（4 项加权）：
- `momentum`：宽基指数（默认上证指数）近 20 日涨幅 ×500
- `limitup`：涨停/跌停比 ×50
- `retailin`：散户净流入占比 ×200
- `overbought`：高于 20 日均线幅度 ×500

### 波动率分位窗口的经验（实测，2026-07 复盘）

- 本周(2026-07-06~10) 用 260 日窗口：vol_pct 恒为 **92%**；改用 60 日窗口后降到 **83–88%**，但 XXFI 仅降约 2 点/天，整体仍偏高。
- **结论**：高波动率分位不是长窗口造成的假象——近 60 天市场确实持续高波动（两窗口都 83%+），属真实信号。若只想看"近期情绪相对冷热"，用 `vol_window=60`；若想看"相对过去一年的极端程度"，用默认 260。两者结论方向一致，仅绝对水位略有差异。
- 回撤 / 动量 / 均线偏离仍固定取近 20 日（标准做法，不随窗口变）。

## 三、何时使用

- 用户问"现在市场是不是恐慌/贪婪到极致了？该抄底还是逃顶？"
- 用户想用一个简单的 0-100 情绪分辅助判断仓位。
- 复盘或教学"散户反向指标"行为模式。

## 四、使用步骤（由调用本技能的 Agent 执行）

### Step 1 — 采集市场数据

> **双模式数据来源（v1.1.6 新增）**：本技能支持两套取数，共用同一套 `compute()` 与权重，结论口径一致。
> - **A. 通达信模式（交互 / 历史复盘）**：用 `tdx-connector` 取上证指数日K（每行自带历史 `up`/`down` 广度）+ akshare 取涨跌停池。支持**历史周度复盘**（见下文"实时自动播报"交互用法）。
> - **B. 纯 akshare 模式（CI / 实时）**：`fetch_market_akshare.py` 直接联网取数，无需 tdx。`stock_zh_index_daily(sh000001)` 算指数分量 + `stock_market_activity_legu()` 拿**当日**广度（上涨/下跌/涨停/跌停家数，一次到位）。⚠️ akshare 广度只返**最新交易日**，故该模式产出"实时当日"指数，不做历史回看。已开源到 GitHub，由 Actions 每个交易日 09:30 自动跑（详见 SKILL.md 末尾"GitHub Actions 实时模式"）。

优先用 **tdx-connector** 或 **westock-data / neodata** 获取以下字段（单位见 `calibration.json` 的 `input_fields`）：

1. **上证指数（代码 000001，setcode=1；也可用沪深300 000300）近 30 日日 K**（tdx_kline，period="4"）：
   - `drawdown` = 近 20 日收盘价最大回撤（负小数）
   - `ret20` = 近 20 日累计涨幅
   - `above_ma20` = (最新收盘 − 20 日均线) / 20 日均线
   - `vol_pct` = 当前 20 日波动率在其"近 N 日"波动率分布中的分位（0-1）。**N 默认 260（全年）**；做情绪复盘/周度回顾想看"纯情绪"时设 `vol_window=60`（仅看近 60 日相对水平，剔除长期低波动基线抬升）。`run_xxfi.py` 已支持 `--vol_window`，如 `python run_xxfi.py --hs300 <路径> --vol_window 60 ...`
2. **涨跌家数 / 涨跌停家数**：
   - `up` `down` `limit_up` `limit_down`
   - ⚠️ **重要发现（已实测）**：`tdx_kline` 拉**指数日 K**（如上证 000001）时，返回的每个交易日 bar 里**自带 `up` / `down` 字段 = 当日上涨/下跌家数**（经核验为**沪市广度**，量级与 akshare 全市场同日值吻合"沪市独占"）。这意味着**逐日历史广度可直接从 tdx 指数日 K 取得，无需 tdx_screener（screener 仅实时、无历史日期参数）**。做周度/历史复盘时优先用这个字段；`limit_up`/`limit_down` 仍用 akshare `stock_zt_pool_em` / `stock_zt_pool_dtgc_em` 按日取（tdx 指数 bar 不含涨跌停计数）。
3. **散户(小单)资金净流入占比** `retail_net`（负为净流出，westock 资金流向接口）

> 港股个股（tdx-connector 不含港股行情）用 **akshare-stock 技能** 取数：`python scripts/ak_query.py hk_history --symbol <0XXXXX> --start YYYYMMDD --end YYYYMMDD`（脚本默认走**新浪源** `ak.stock_hk_daily`，东方财富源本机常断连已弃用，切勿默认使用）。已核验代码：胜宏科技 **02476**、澜起科技 **6809**、开拓药业-B **09939**。

### Step 2 — 计算指数
把采集到的字段以 JSON 传入脚本：

```bash
python xiaoxu_fear_index.py --json '{"drawdown":-0.08,"ret20":0.02,"above_ma20":0.01,"up":1800,"down":3200,"limit_up":40,"limit_down":30,"vol_pct":0.78,"retail_net":-0.02}'
```

或逐项传参：
```bash
python xiaoxu_fear_index.py --drawdown -0.08 --ret20 0.02 --above_ma20 0.01 --up 1800 --down 3200 --limit_up 40 --limit_down 30 --vol_pct 0.78 --retail_net -0.02
```

自检可用 `--demo`（内置恐慌/贪婪两个校准样例，应分别输出 BUY / SELL）。

### Step 3 — 解读并给结论
将脚本输出的 `XXFI` / `GreedIndex` / `contrarian_signal` / `level` / `advice` 汇报给用户，并对照 `calibration.json` 的 `interpretation_bands` 给出可执行建议（低吸 / 持有 / 减仓）。

## 五、解读区间速查

| XXFI | 市场状态 | 反向信号 | 建议 |
|---|---|---|---|
| ≥75 | 极度恐惧（小旭式恐慌割肉区） | BUY | 分批低吸，历史卖后常反弹 +9%~+24% |
| 60–75 | 恐惧（她倾向割肉） | ACCUMULATE | 逢低吸纳，不跟风杀跌 |
| 40–60 | 中性 | HOLD | 按自身策略持有 |
| 25–40 且贪婪高 | 贪婪（她倾向追高） | REDUCE | 逢高减仓，不追涨 |
| <25 且贪婪高 | 极度贪婪（小旭式追涨山顶区） | SELL | 减仓避险，历史买后常回落 −12%~−21% |

## 五、实时自动播报（开盘自动化）

把"取数→算指数→出报告→回传"串成一条自动流水线，用 `run_xxfi.py` 作为执行入口：

```bash
python run_xxfi.py --hs300 <指数日K文件路径> \
                   --breadth '{"up":2400,"down":2600,"limit_up":55,"limit_down":25,"retail_net":-0.01}' \
                   --out <报告输出目录>
```

### 取数（由调用方用 tdx-connector 完成）
1. **指数分量**：`tdx_kline`（code=000001, setcode=1, period="4", wantNum=260, tqFlag="1"）→ 保存文件，`run_xxfi.py --hs300 --vol_window 260`（默认）自动算出 回撤/动量/均线偏离/波动率分位；若想看"纯情绪"相对水平，改 `--vol_window 60`。
2. **盘面广度**：`tdx_quotes` 查 上证指数(000001) 实时行情，提取 涨家数/跌家数/涨停数/跌停数（多数行情源含这些字段）；若缺失，用 `tdx_screener` 以"今日涨停"/"今日跌停"(limit=200) 计数兜底；`retail_net`（散户净流入占比）可经 `tdx_indicator_select` 或资金流接口获取，没有则置 0。
3. 广度缺失时 `run_xxfi.py` 自动降级为"指数版 XXFI"（仅指数派生分量），报告标注置信度较低。

### 自动化配置（WorkBuddy 定时任务）
- 任务名建议：小旭恐惧指数·开盘播报
- 频率：每个交易日 09:30（`FREQ=WEEKLY;BYDAY=MO,TU,WE,TH,FR;BYHOUR=9;BYMINUTE=30`，节假日不自动剔除）
- 任务提示词（prompt）要点：
  1. 用 `tdx_kline` 拉上证指数日K（代码 000001，260根）并记录文件路径；
  2. 用 `tdx_quotes` 取上证指数实时广度（涨/跌家数、涨/跌停），组装 breadth JSON；
  3. 运行 `python run_xxfi.py --hs300 <路径> --breadth '<json>' --out <项目目录>`；
  4. 用 `present_files` 展示生成的 `xxfi_report.md`；
  5. 输出一句中文结论（XXFI 数值 + 反向信号）。
- **回传小程序**：在自动化任务设置中开启「推送到小程序」，开盘播报会自动同步到微信小程序（本机远程）。交互式会话里的产物不会自动推，需手动"一键回传"。

### GitHub Actions 实时模式（纯 akshare，开源自动化）

技能已开源，Actions 每个交易日 **09:30 北京时间**自动跑"取数→算→出报告→提交"，产物在仓库 `output/`：

- `output/xxfi_report.md` / `xxfi_report.json`：当日报告与结构化结果
- `output/history.jsonl`：每日一行 `{date, xxfi, greed, signal, level}` 历史

```bash
# 本地复现（需 pip install -r requirements.txt）
python run_xxfi.py --akshare --vol_window 60 --out output
```

- 工作流：`.github/workflows/xxfi-daily.yml`（cron `30 1 * * 1-5` = 01:30 UTC 周一至五；支持手动 `workflow_dispatch`）。
- 防污染：若 `legu` 返回的数据日期 ≠ 当天（休市/半日市），跳过写历史。
- 波动率窗口默认 `60`（纯近期情绪）；想看相对全年极端度改 `--vol_window 260`。
- 取数兜底：`legu` 失败时退用 `stock_zt_pool_em` / `stock_zt_pool_dtgc_em` 计数。

> 该模式产出的"实时当日"指数与通达信交互模式的"历史周度复盘"互补，二者共用 `compute()`，口径一致。

- 原始操作记录来自用户提供的 `小旭操作实录.txt` 及后续补充聊天记录；部分未具名股票（如"某股""医药"）无法核验，已从统计中剔除。

- 50% 卖飞率部分受 2025-2026 牛市 beta 影响，"放大了概率"属用户自述，使用时请结合当下市场环境修正先验。
